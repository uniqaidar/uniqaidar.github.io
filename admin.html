🚨 Google Recovery — چارەسەری "Couldn't Fetch"🚨 Google Recovery — Fix "Couldn't Fetch"

All these actions are ranked by importance. Do each one in order and wait a few minutes between each step.

① Live sitemap.xml Audit (HTTP Audit)

Connects directly to your site and checks Content-Type, HTTP Status, and first byte — exactly what Googlebot sees.

② Delete & Resubmit sitemap in Google Search Console

This is the most powerful action. Google discards its old "Couldn't fetch" cache and re-reads from scratch.

1. Go to Google Search Console → Sitemaps ↗
2. Click the ⋮ menu next to /sitemap.xml and Remove
3. Wait 30 seconds
4. In the "Add a new sitemap" box, type: sitemap.xml and click SUBMIT
5. Also submit: sitemap_index.xml (add both entries)
6. Wait 24 hours
③ URL Inspection — Manual Indexing Request

This is the second most powerful tool. Google can fetch the URL individually, treating it as a priority.

1. Go to URL Inspection Tool ↗
2. Enter the sitemap URL: https://uniqaidar.pages.dev/sitemap.xml
3. Click "Test Live URL" — this shows if Google can fetch it
4. If successful, click "Request Indexing"
④ Purge Cloudflare Pages Cache

Cloudflare may be serving a cached old version of sitemap.xml. Purging the cache ensures Googlebot sees the latest file.

1. Go to Cloudflare Dashboard ↗
2. Select: Pages → uniqaidar → Deployments
3. Click on the latest deployment → "Purge Cache" or re-deploy
4. If using a custom domain, go to Caching → Cache Rules and clear sitemap.xml cache
⑤ Googlebot Simulation — User-Agent Check

Tests whether your site blocks Googlebot. If this tool cannot read the sitemap — neither can Google.

⑥ Validate XML — Self-check sitemap

Downloads your current sitemap.xml and checks it for XML errors, invalid URLs, and illegal characters.

⑦ IndexNow — Instant Indexing for Bing / Yandex / Naver

IndexNow is the instant indexing protocol for Bing, Yandex, Naver, and Amazon Bot. One ping — all engines are notified. Google has not yet adopted it, but may in the future.

These 3 steps are required only once:
1. Click "Generate New Key" or enter your own Bing key
2. Click "Save Key File" — creates a file in GitHub: /{key}.txt
3. Do this once — then "Ping IndexNow" runs automatically on every Atomic Save
⑧ Bing Webmaster Tools — Submit sitemap to Bing

Bing manages sitemaps differently from Google. Enter your Bing Webmaster Tools API Key to submit directly.
Get your key: Bing Webmaster Tools ↗ → Settings → API Access → Generate API Key

If you don't have an API Key, do it manually:
1. Go to Bing Webmaster → Sitemaps ↗
2. Click "Add Sitemap"
3. Enter: https://uniqaidar.pages.dev/sitemap.xml
4. Also add: https://uniqaidar.pages.dev/sitemap_index.xml
